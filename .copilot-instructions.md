Copilot — Generate fully SEO-optimized landing pages and prepare the entire PureText project for correct indexing, metadata, and authority building. Follow these rules exactly:

-----------------------
SEO LANDING PAGE RULES
-----------------------

1. Each SEO page MUST be a silent landing page. It must NOT appear in the site UI, navbar, buttons, or main layout.

2. Use clean minimal HTML structure:
   - no animations
   - no shadcn/ui components
   - no imports from the main app
   - React component with static content only
   - no logic, no hooks, no state

3. Add full SEO metadata for each page:
   - <title> using the page’s primary keyword
   - <meta name="description">
   - Open Graph tags (og:title, og:description, og:url, og:image)
   - canonical link to the correct SEO URL

4. Add 600–1500 words of SEO content:
   - H1 with the main keyword (e.g., “ProtectedText Alternative”)
   - H2 and H3 sections explaining:
       * what the topic means
       * benefits
       * use cases
       * privacy suitability
       * why PureText is faster, cleaner, no-signup, privacy-first
   - Add a simple comparison table between PureText and the competitor/tool
   - End with a soft CTA:
       “Use PureText → https://puretext.me”
     (NO buttons, no styled elements)

5. These pages must be built for Google indexing, not UX visuals.

6. Keep layout semantic:
   - <main>
   - <header>
   - <section>
   - <article>
   - <footer>

7. Do not touch or modify any existing UI of the app.

8. Create a separate SEO page for each route:
   - /protectedtext-alternative
   - /best-protectedtext-alternatives
   - /puretext-vs-protectedtext
   - /plain-text-online-editor
   - /remove-formatting-online
   - /paste-as-plain-text-tool

-----------------------
SITE-WIDE SEO RULES
-----------------------

1. Generate a sitemap.xml including ALL SEO landing pages AND the homepage.

2. Create or update robots.txt with:
   - Allow: /
   - Sitemap: https://puretext.me/sitemap.xml

3. Ensure every SEO page includes a canonical URL to prevent duplicates.

4. Add global Open Graph defaults in the main HTML <head>:
   - <meta property="og:site_name" content="PureText">
   - <meta property="og:type" content="website">
   - <meta property="og:image" content="/og-image.png">

5. Each SEO page must export its own metadata head element without interfering with any other page.

6. Vite build must correctly output metadata for each static route in /dist.

7. Add font preload links (Inter or Geist) for performance.

8. Ensure <html lang="en"> is present for accessibility + SEO.

- Ensure these pages rank independently in Google.
- Build perfect indexing via sitemap and robots.
